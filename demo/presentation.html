BttrLazyLoading

Responsive Lazy Loading plugin for JQuery.

Download the development version Add an issue

BttrLazyLoading is a Jquery plugin that allows your web application to defer image loading until images are scrolled to but not only! BttrLazyLoading also allows you to have different version of an image for 4 differents screen sizes: phones (<768px), tablets (≥768px), desktops (≥992px) and large Desktops (≥1200px).

Plugin dependencies: BttrLazyLoading depends on jQuery (this means jQuery must be included before the plugin files).

Options

Options can be set for one or several images at the same time or for the whole page at once (using the setOptions global function)

Name type default description
delay integer 0

Adds delay to the image loading time.
threshold integer 0

By default images are loaded when they appear on the screen. If you want images to load earlier use threshold parameter. Setting threshold to 200 causes image to load 200 pixels before it appears on viewport.
animation string "bounceIn"

Adds an animation when the image loads. You have access to all of the following animation created by Animate.css or use Null if you do not want any animation

['flipInX', 'flipInY', 'fadeIn', 'fadeInUp', 'fadeInDown', 'fadeInLeft', 'fadeInRight', 'fadeInUpBig', 'fadeInDownBig', 'fadeInLeftBig', 'fadeInRightBig', 'slideInDown', 'slideInLeft', 'slideInRight', 'bounceIn', 'bounceInDown', 'bounceInUp', 'bounceInLeft', 'bounceInRight', 'rotateIn', 'rotateInDownLeft', 'rotateInDownRight', 'rotateInUpLeft', 'rotateInUpRight', 'lightSpeedIn', 'rollIn']
event string "scroll"

Defines the event that will be use to trigger the image loading/updating.
placeholder string (image base64) "data:image/gif;base64..."

Base 64 image that is used when the image loads.
container string (CSS selector) window

You can also use this plugin for images inside scrolling container, such as div with scrollbar. By default the container is the window.
retina boolean false

Enable a better quality on Retina screens.

BttrLazyLoading uses a naming convention @2x to display retina's images. BttrLazyLoading will therefore seek for "yourImage@2x.gif" on retina' screens instead of "yourImage.gif"
triggermanually boolean false

Whether or not to trigger the first image load manually.
updatemanually boolean false

Whether or not to trigger the image update (needed when the window resizes for example) manually.
onBeforeLoad function($image, Object inst) null

Callback called just before that the image loads.
onAfterLoad function($image, Object inst) null

Callback called just after that the image loads.
onError function($image, Object inst) null

Callback called when the plugin cannot find any image src that exist.

Set options on Instanciation

$("#yourImageId").bttrlazyloading({
	img: {
		xs: {
			src: "img/720x200.gif",
			width: 720,
			height: 200
		},
		sm: {
			src: "img/360x200.gif",
			width: 360,
			height: 200
		},
		md: {
			src: "img/470x200.gif",
			width: 470,
			height: 200
		},
		lg: {
			src: "img/570x200.gif",
			width: 570,
			height: 200
		}
	},
	retina: true,
	transition: 'fadeInUp',
	delay: 1000,
	event: 'click',
	container: 'document.body',
	threshold: 666,
	placeholder: 'test',
	onBeforeLoad: function($img, bttrLazyLoading) {
	},
	onAfterLoad: function($img, bttrLazyLoading) {
	},
	onError: function($img, bttrLazyLoading) {
	}
})

onError

BttrLazyLoading will try to load any of the given image source you define. If an image does not exist an error event will be triggered and if none are found the onError function as well as the bttrlazyloading.error event will be triggered.

<img id="yourImageId" class="bttrlazyloading"
	data-bttrlazyloading-xs-src="imageDoesNotExistXs.jpg"
	data-bttrlazyloading-sm-src="imageDoesNotExistSm.jpg"
	data-bttrlazyloading-md-src="imageDoesNotExistMd.jpg"
	data-bttrlazyloading-lg-src="imageDoesNotExistLg.jpg"
/>
$('#yourImageId').bttrlazyloading({
	onError: function($img, bttrLazyLoading) {
		$img.remove();
	}
});

Ranges

Ranges define the size of your images for several type of screens. They can only be overwritten for the whole page (using the setRanges global function)

Name type default description
ranges object 
{
 'xs' : 767,
 'sm' : 768,
 'md' : 992,
 'lg' : 1200
 }

Ranges follows Bootstrap 3 grid sizes. xs for phones (<768px), sm for small devices Tablets (≥768px), md for medium devices Desktops (≥992px) and lg for large devices Desktops (≥1200px).

For most needs, the default values (Bootstrap 3 grid) is enough.

Global methods

The BttrLazyLoading plugin does not have any public methods. Nevertheless two global methods are available to interact with all BttrLazyLoading instances in the page.

Returns: bttrlazyloading Global

setRanges()

$.bttrlazyloading.setRanges({
	xs: 767,
	sm: 768,
	md: 992,
	lg: 1200
});
Returns: bttrlazyloading Global

setOptions()

Events

Event Type description
bttrlazyloading.beforeLoad

This event is triggered just before the "bttrlazyloading.load" event.

More information in the option section: onBeforeLoad
bttrlazyloading.load

This event is triggered when the image loading is triggered. You can trigger it manually like this: 

$('#yourImageId').trigger('bttrlazyloading.load');

More information in the option section: triggermanually
bttrlazyloading.afterLoad

This event is triggered just after the "bttrlazyloading.load" event.

More information in the option section: onAfterLoad
bttrlazyloading.error

This event is triggered when none of the images (xs, sm, md and lg) exist. The classic "error" event could therefore be triggered up to 8 times (4 times for a normal screen and 8 times for a retina screen) while "bttrlazyloading.error" will be triggered only once.

More information in the option section: onError

Tips

Only one image size needed!

BttrLazyLoading always try to load the biggest version of the image available. Therefore the following example will work on every screen too.

<img id="yourImageId" class="bttrlazyloading"
	data-bttrlazyloading-md-src="img/455x350.gif"
/>

No src attribute?

Browsers are too quick for us! In order to provide the fastest loading time possible, browsers preload all of the images that they can identify

Choosing A Responsive Image Solution, Smashing Magazine

You may have noticed that none of our image elements have a "src" attribute. Well that is normal! As describe here by the Smashing Magazine, the browsers preload the images even before the document ready event which make it impossible for this plugin to process the information before the image loads.

Fallback for Non JavaScript Browsers

<img class="bttrlazyloading"
	data-bttrlazyloading-xs-src="img/768x200.gif"
	data-bttrlazyloading-xs-width="768"
	data-bttrlazyloading-xs-height="200"
/>
<noscript>
<img class="bttrlazyloading" src="img/768x200.gif" width="768" height="200"/>
</noscript>

Convenient way to set data attributes

The following examples are equivalent:

	data-bttrlazyloading-xs-width="768"
	data-bttrlazyloading-xs-height="200"
/>

	data-bttrlazyloading-xs='{"src": "img/768x200.gif", "width" : 768,  "height" : 200}'
